搭载Github+hexo博客记录

Date: 四月 18, 2020  9:47:38 Category: Blog

首先：注册Github账号 + 创建Github仓库 + 购买域名，开搞！

注： 仓库名最好是，用户名.github.io

1.搭载hexo环境

###1.1 Git安装
###1.2 Node.js+npm安装
注：打开cmd输入一下检测是否安装成功

git --version
node -v
npm -v

###1.3 配置ssh免密登录
打开Git Bash，配置用户信息

git config --global user.name 自己的Github用户名
git config --global user.email 自己的注册邮箱

生成此账户的ssh密钥

ssh-keygen -t rsa -C 自己的注册邮箱

连续三个回车
找到生成的.ssh文件，复制id_rsa.pub中的内容
打开Github账户的的仓库设置密钥界面，Title随意，将内容复制到Key中即可

检测是否免密成功，打开Git Bash输入

ssh git@github.com

出现类“successfully authenticated”即可。

2.安装并初始化hexo

在本地创建一个文件夹命名Blog，存放博客框架以及未来要发布的网页；

单机文件夹，按住shift键，打开shell，如此即可在此目录下进行操作

npm install -g hexo-cli #安装hexo
npm install hexo-deployer-git --save #安装Git部署插件
hexo init 你刚创建的博客文件名 #初始化

可先预览
hexo g #生成静态文件
hexo s #生成本地服务器，用于预览

打开浏览器，输入localhost:4000，就是雏形。

3.映射hexo与Github仓库

上一步只能在本地预览博客，想要被其他人看到，就要告知hexo，博客的具体位置，因为把博客托管在github中，因此，建立它们的映射关系；

打开自己建的博客文件，找到_config.yml配置文件，拉到最底下，在repo后填入你Github仓库的路径，最后加上.git，记得最前面还有个空格,保存。

映射关系成功建立，可以通过hexo来部署博客了，

hexo clean #清楚缓存文件，防止更新不及时
hexo d -g  #生成静态文件，并部署配置文件

在浏览器，输入 用户名.github.io ，我的博客可以在网络上被访问啦。

4.绑定个性化域名

打开阿里云，进入控制台，点击域名解析，添加记录，
192.30.252.153是Github的地址；

这个的主机记录，我写了blog，那么在网页上找我的博客，输入blog.superfwithy.com即可，可以写其他自己喜欢的。
进入仓库，找到Setting，修改domian，

进入本地博客的文件夹，在source目录下，创建一个CNAME的记事本文件，输入自己的域名，保存。

老样子，单机博客文件夹，按住shift，打开shell，

hexo clean
hexo d -g

重新部署后，我就可以在浏览器上直接输blog.superfwithy.com进自己博客拉！

5.下载必要的插件

我发博的时候吧，插入本地图片的问题卡了蛮久的，要么md里都没有，要么浏览器里没有，最后总结，得下载一个插件，这个插件允许应用本地图片，

首先得将站点的配置文件打开，还是writing部分，将post-asset-folder的值修改为true，这是博客的资源文件夹，意思就是当你创建一个新博客时，会自动创建与博客同名的文件夹，可以把和博客相关的资源都放在这个文件夹里，通过相对路径来引用,我还比较喜欢图片居中，width可以调节图片大小，因此，html标签如下；

<div align=center>
<img width=500 src=同名文件夹/图片>
</div>
